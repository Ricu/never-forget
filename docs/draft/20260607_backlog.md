## Person

Represents an individual referenced by memories. Is also structured to be accessible in a "contacts" style.

Fields:

* id
* name
* aliases
* created_at
* updated_at
* More later, like e.g. date of birth, adress, phone number etc.


## Time Travel

Have a global time control to go back to a specific point in time.

## Detect bad audio/transcript early

Have a mechanism that detects if the input is non-sense for some reason (e.g. bad audio leading to bad transcripts)
